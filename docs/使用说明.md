# WDS 图包管理器 — 完整使用说明

版本 1.1.0 | 适用于 WDS 系列兵棋游戏（Panzer Campaigns / Panzer Battles 等）

---

## 这是什么

一个命令行工具，自动化管理 WDS 游戏的美化包（图包）。以前你需要手动备份、复制、粘贴数百个 BMP 文件才能装一个美化包，现在一条命令搞定，而且随时可以一键还原。

## 安装

双击项目文件夹里的 `install.bat`，等它跑完就行。装好后在任意 cmd / PowerShell / 资源管理器地址栏输入 `wds` 即可启动。

如果安装失败，确认：
1. 已安装 Python 3.10+（命令行输入 `python --version` 检查）
2. Python 已加入系统 PATH

## 启动

输入 `wds` 回车，进入交互界面：

```
  wds>
```

在提示符后直接输入命令，不需要再加 `wds` 前缀。输入 `q` 或 `exit` 退出。

也可以不进入交互模式，直接执行单条命令（如 `wds scan "路径"`），执行完自动回到系统提示符。

## 命令详解

### scan — 扫描预览

```
wds> scan "D:\Mods\Hawkeyes_P39_Mod_V2"
```

只读操作，不修改任何文件。展示 mod 中每个 BMP 文件将被安装到游戏的哪个位置，以及匹配置信度（自动 / 歧义 / 无匹配）。

可选参数：`-g P39` 手动指定目标游戏（不指定则自动推断）。

### install — 安装美化包

```
wds> install "D:\Mods\Hawkeyes_P39_Mod_V2"
```

交互式安装：工具会按目录组逐组展示匹配结果，每组你可以：
- **回车** — 接受建议的目标路径
- **e** — 手动修改目标路径
- **v** — 展开查看组内所有文件的匹配详情
- **s** — 跳过此组（不安装这些文件）
- **q** — 退出安装

可选参数：
- `--auto` — 跳过交互审查，直接使用自动匹配结果
- `-g P39` — 手动指定目标游戏
- `-n "鹰眼波兰包"` — 设置美化包别名

注意：美化包需先解压为文件夹，暂不支持直接读取 zip/7z/rar。

### status — 查看状态

```
wds> status P39       查看某个游戏
wds> status           查看所有游戏汇总
```

显示每个美化包的状态：■ 已安装 / ◐ 部分激活 / □ 未安装。

### uninstall — 卸载美化包

```
wds> uninstall hawkeyes_p39_mod_v2
```

将该 mod 替换过的所有文件还原为原版。mod 的注册信息保留（可以 switch --on 重新启用）。

### switch — 启用/禁用

```
wds> switch hawkeyes_p39_mod_v2 --off    禁用（还原原版）
wds> switch hawkeyes_p39_mod_v2 --on     启用（重新应用 mod 文件）
```

不删除任何数据，只是切换激活状态。适合"想暂时看看原版效果"的场景。

### rename — 修改别名

```
wds> rename hawkeyes_p39_mod_v2 "鹰眼波兰39"
```

### ls-mods — 列出所有美化包

```
wds> ls-mods
wds> ls-mods --all    含无 mod 的游戏
```

### undo — 回退上一步

```
wds> undo
wds> undo -g P39     指定游戏
```

撤销最近一次操作：install 的 undo 是卸载，uninstall 的 undo 是重新启用，switch 的 undo 是反向切换。每次只能回退一步。

## 工具管理

这些命令在交互模式外使用：

```cmd
wds -u          更新到最新版本（从 GitHub 拉取）
wds -un         卸载本工具（会询问是否先还原所有美化包）
wds -help       打开本说明文档
wds --version   显示版本号
```

## 备份与安全

所有操作均安全可逆。工具在游戏目录内创建 `_backup/` 文件夹存放备份和状态数据：

- `{游戏ID}_original/` — 原版文件全量备份（首次安装时自动创建，之后不再变动）
- `{mod_id}_{时间戳}/` — 安装前快照（多 mod 叠加时保证回退链正确）
- `{mod_id}_modfiles/` — mod 文件副本（供 switch --on 重新启用）
- `game_registry.json` — 注册表，记录每个文件被哪个 mod 替换
- `operation_log.json` — 操作日志，供 undo 使用

游戏 exe 完全忽略 `_backup/` 目录（已实证），不影响游戏运行。

**重要：请勿手动删除 `_backup/` 文件夹，否则无法回退。**

## 游戏根目录（WDS_ROOT）

工具需要知道你的 WDS 游戏装在哪里。它按以下优先级确定路径：

1. **命令行参数**（一次性）— 在任意命令后加 `--wds-root`：
   ```cmd
   wds install "D:\Mods\hawkeyes_p39" --wds-root "E:\Games\WDS"
   wds status --wds-root "E:\Games\WDS"
   ```
   只对这一次命令生效。

2. **环境变量**（永久生效）— 设置一次，之后所有命令自动使用：
   ```cmd
   setx WDS_ROOT "E:\Games\WDS"
   ```
   设置后需要**新开一个终端窗口**才生效（当前窗口不会立即刷新）。

3. **默认值** — 如果以上都没设，工具默认去找 `D:\WDS`。

**游戏文件夹搬家了怎么办？**

比如你把 `D:\WDS` 整个剪切到了 `E:\Games\WDS`，只需要更新环境变量：

```cmd
setx WDS_ROOT "E:\Games\WDS"
```

不需要重新安装工具，也不需要重新安装美化包——`_backup/` 文件夹跟着游戏目录一起搬走了，所有备份和注册表都还在。

如果你的游戏本来就在 `D:\WDS`，这一节可以完全忽略。

## 常见问题

**Q: 安装后游戏里看不到变化？**
确认游戏 exe 已完全关闭再重新打开。部分 WDS 游戏有缓存。

**Q: mod 是 zip 格式怎么办？**
先解压为文件夹，再对文件夹执行 install。

**Q: 想装多个美化包会冲突吗？**
不会。后装的 mod 会覆盖先装 mod 的同名文件，但备份链完整保留。用 switch 可以随时切换哪个 mod 生效。

**Q: 误操作了怎么办？**
`wds undo` 回退上一步。或者 `wds uninstall <mod_id>` 还原原版。最坏情况：`_backup/{游戏ID}_original/` 里永远有完整原版。

**Q: 怎么彻底删除工具？**
`wds -un`，会先问你是否还原所有美化包，然后卸载工具本身。
